Final considerations

The best possible method involves using real domains or fake domains for fronting (I still can’t figure out how we can create this to remain undetectable), along with simple forwarding from internet exchanges connected to countries with blocking. This approach can effectively help bypass restrictions.

For example, in Iran, we have a direct connection to Delta-Telecom as an upstream provider. Now, platforms that are blocked, VPN providers, or any kind of service can contact these companies to establish a contract for IP forwarding. This would enable the rerouting of traffic to the correct origin.

What I mean is that if I want to access tor bridge and the IP is blocked in my country, I can use any IP that BGP routes see as the best option and prefer it to the upstream provider with whom the site has a contract. For example, in Iran, we use one of the IP ranges associated with the upstream network, Delta-Telecom.

When packets are received from Iran at Delta-Telecom, the destination address can be modified to the correct origin IP based on the user's source IP.

If we want to simplify this process, we can think of it as a large IP forwarding operation at the upstream level, where traffic is routed based on the source IP to the correct destination, like tor bridge. Now, tor bridge can send data back to us using IP spoofing.

It may sound easy, but when it comes to upstream companies, it’s quite complex. However, this approach can provide a significant advantage, making it difficult to detect or block.
